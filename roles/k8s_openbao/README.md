# k8s_openbao

Deploy and operate OpenBao runtime for `rtr-fabric`.

## Scope

- deploy OpenBao in `dev` or `raft` mode
- enforce production safety checks (HTTPS/TLS, HA profile, no insecure LDAP mode)
- bootstrap AppRole auth and local credential files used by automation
- seed selected secret paths from local inventory secrets into OpenBao

## Main inputs

- `k8s_openbao_runtime_mode`: `dev` or `raft`
- `k8s_openbao_runtime_replicas`: raft replica count
- `k8s_openbao_runtime_allow_single_node_prod`: explicit prod exception gate
- `openbao_url`, `openbao_validate_certs`, `openbao_auth_method`
- `openbao_approle_role_id_file`, `openbao_approle_secret_id_file`
- `k8s_openbao_runtime_rotate_approle_secret_id`: rotate AppRole secret id on each run when `true`

Seeding toggles:

- `k8s_openbao_runtime_seed_net_services`
- `k8s_openbao_runtime_seed_openwrt_wifi_edge_pre`
- `k8s_openbao_runtime_seed_acme`
- `k8s_openbao_runtime_seed_ca`
- `k8s_openbao_runtime_seed_ops_etcd_offsite`
- `k8s_openbao_runtime_seed_github_actions_runner`
- `k8s_openbao_runtime_seed_bgp`
- `k8s_openbao_runtime_bgp_neighbor_password_file` (local bootstrap file used when BGP secret path is empty)

## Outputs and artifacts

- runtime manifests applied in namespace `k8s_openbao_runtime_namespace`
- local files (controller node):
  - `k8s_openbao_runtime_root_token_file`
  - `k8s_openbao_runtime_read_token_file`
  - `openbao_approle_role_id_file`
  - `openbao_approle_secret_id_file`

## Idempotence notes

- KV payload writes happen only when target payload differs.
- AppRole role_id remains stable.
- AppRole secret_id is not regenerated by default (`k8s_openbao_runtime_rotate_approle_secret_id=false`).
  Set it to `true` for forced rotation.
- BGP seed writes only when needed:
  - reuse existing OpenBao `neighbor_password` if already present
  - otherwise use explicit local/vault input
  - otherwise generate once into `k8s_openbao_runtime_bgp_neighbor_password_file` and seed OpenBao

## Failure modes

- missing or unsafe prod settings fail early with `assert`
- missing OpenBao init bundle in already-initialized raft mode fails to avoid accidental re-init
- invalid LDAP transport profile in prod fails to avoid insecure auth exposure
